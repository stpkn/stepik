## Стратегия реализации

### Подход
Инкрементальный и contract-first: сначала стабилизируется авторизация (P1), затем добавляются пользовательские учебные сценарии и аналитика (P2/P3).

### Порядок выполнения
1. Фаза 1: `step-01-auth-persistence-foundation`

### Критерии завершения
План считается завершенным, когда все функции P1/P2/P3 из `.pldf/concept.md` покрыты запланированными шагами с явными зависимостями и тестами.

## Тестирование

### Стратегия тестирования
Для каждого шага задаются unit/integration проверки backend и минимум один ручной E2E-сценарий из пользовательского потока.

### Типы тестов
- **Unit тесты**: сервисы валидации и репозитории
- **Integration тесты**: API endpoints по OpenAPI-контракту
- **E2E тесты**: ручной или автоматизированный сценарий входа и перехода в защищенный раздел

### Автоматизация
Backend тесты запускаются через `pytest`, фронтенд-проверки сценариев входа подготавливаются для `vitest`/`playwright`.
