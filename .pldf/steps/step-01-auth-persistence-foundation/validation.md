# Критерии валидации шага 01: Основа авторизации и SQLite persistence

## Критерии успешного завершения

### Функциональные требования

**Backend:**
- [ ] Реализован endpoint `POST /login` в соответствии с `.pldf/contracts/openapi.yaml`
- [ ] Авторизация проверяет учетные данные через SQLite (`users`)
- [ ] Неверные данные возвращают предсказуемую ошибку 401

**Frontend:**
- [ ] Форма входа отправляет `login` и `password` в ожидаемом формате
- [ ] Успешный ответ backend переводит пользователя в защищенный раздел
- [ ] Ошибка авторизации отображается в UI без поломки экрана

**Интеграция:**
- [ ] Ответы API совпадают с контрактом `.pldf/contracts/openapi.yaml`
- [ ] Поток login -> protected route проходит end-to-end

### Технические требования

**Backend:**
- [ ] Код следует принципам SOLID
- [ ] Используется согласованная структура модулей проекта
- [ ] Модели данных соответствуют `.pldf/data-model.md`
- [ ] Обработка ошибок согласована со стандартами FastAPI

**Frontend:**
- [ ] Код следует принципам SOLID
- [ ] Не нарушены границы feature-first и существующая структура
- [ ] State management для auth работает корректно
- [ ] HTTP клиент отправляет запрос в backend по текущей конфигурации API
- [ ] Обработка ошибок реализована предсказуемо для пользователя

### Тестирование
