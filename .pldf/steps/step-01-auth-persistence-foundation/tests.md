# Шаг 01: Тесты

**Номер**: step-01-auth-persistence-foundation  
**Приоритет**: P1

## Автоматические тесты

### Backend (pytest)

- Проверка успешного входа по корректным данным в SQLite.
- Проверка ошибки 401 при неверном пароле.
- Проверка ошибки 401 при несуществующем `login`.
- Проверка соответствия структуры ответа контракту `.pldf/contracts/openapi.yaml`.

**Как запустить**:
```bash
cd backend && pytest -q
```

**Ожидаемый результат**: все auth-тесты проходят, форматы ответов совпадают с контрактом.

### Frontend (vitest, при наличии)

- Проверка отправки формы с полями `login` и `password`.
- Проверка отображения ошибки при 401 от backend.

**Как запустить**:
```bash
cd frontend && npm test
```

**Ожидаемый результат**: форма входа корректно реагирует на успех и ошибку.

## Ручные тесты

- [ ] Успешный вход тестовым пользователем -> переход на защищенный маршрут.
- [ ] Неверный пароль -> остается экран входа, показывается сообщение об ошибке.
- [ ] Повторная попытка после ошибки с корректными данными -> успешный вход.
