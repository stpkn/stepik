# Шаг 01: Основа авторизации и SQLite persistence

**Номер**: step-01-auth-persistence-foundation  
**Дата создания**: 2026-04-16  
**Зависимости**: нет

## Описание

Этот шаг формирует минимально устойчивую основу P1-функции входа: один источник истины для контракта авторизации, backend-логики и данных в SQLite.

Почему шаг важен:
- Без него невозможно надежно развивать остальные P1-функции (дашборд, обучение, статистика, активность) на корректном auth-потоке.
- Он закрепляет принцип contract-first из `.pldf/contracts/openapi.yaml`.
- Он связывает UI и API с фактической моделью данных из `.pldf/data-model.md`.

Покрываемая функция из концепции:
- **P1**: Authentication

Связанные артефакты:
- UI: `.pldf/ui-design.md`, `.pldf/ui-prototype/login.html`
- API: `.pldf/contracts/openapi.yaml`, `.pldf/contracts/auth-login.yaml`
- Данные: `.pldf/data-model.md` (сущности `User`, `LoginAttempt`)

## Технические границы

- Backend: только `POST /login` и слой доступа к SQLite для проверки пользователя.
- Frontend: только поля/формат запроса и обработка успешного/ошибочного ответа по контракту.
- Архитектура: не нарушать границы feature-first и не добавлять cross-feature зависимости.
