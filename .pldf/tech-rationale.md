# Детальное обоснование выбора стека (/pldf.tech)

## Что изменилось
Вы указали, что в проекте используется SQLite, поэтому технологический стек обновлен с фиксацией SQLite как основной БД для MVP.

## Почему SQLite подходит сейчас
- Минимальная инфраструктура: не требует отдельного DB-сервера.
- Быстрый локальный запуск: удобно для учебного процесса и итераций.
- Достаточно для домена MVP: users, courses, enrollments, progress, activity.

## Риски и границы
- Ограничения на высокую конкурентную запись.
- Ограниченный горизонтальный рост в сравнении с PostgreSQL.

## Как это влияет на следующий этап (/pldf.architecture)
1. Нужен явный data access layer (repositories/services).
2. Нужна схема БД и миграции (Alembic или эквивалент).
3. Нужна стратегия перехода с mock-данных к SQLite storage.

## Проверка совместимости
- Frontend: React 18 + Router 6, сборка Webpack/Babel.
- Backend: FastAPI + Uvicorn.
- БД: SQLite интегрируется без изменения frontend-контрактов API.

## Вывод
Для текущего масштаба проекта выбор SQLite обоснован и согласован с вашими обновлениями.
